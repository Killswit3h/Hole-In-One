# Golf Swing Analyzer

An Android app that records your golf swing, detects body pose in real-time using on-device AI, and gives you instant coaching feedback — all without any server or internet connection.

---

## Features

### Record Screen
- Full-screen camera preview using the device's back camera
- Tap to start/stop video recording
- While recording, ML Kit Pose Detection runs on every camera frame in real-time, capturing 33 body landmarks 30 times per second
- All detected poses are saved alongside the video automatically

### Analysis Screen
- Plays back your recorded swing video
- A **green skeleton overlay** is drawn on top of the video, synchronized frame-by-frame with playback — shows shoulders, elbows, wrists, hips, knees, and ankles
- **Swing phase label** in the corner updates as video plays: `Setup → Backswing → Impact → Follow-Through`
- **Joint angle panel** shows live values as video plays:
  - Shoulder rotation (tilt of the shoulder line)
  - Hip rotation (tilt of the hip line)
  - Lead elbow bend (how straight your lead arm is)
- **Coaching tips panel** shows up to 3 personalized tips based on your measured angles (e.g. *"Keep your lead arm straighter at impact"*)

### Past Swings Screen
- Lists all previously recorded swings sorted by date, newest first
- Shows recording date, time, and duration
- Tap any swing to re-open full analysis
- Swipe left to delete a recording

### Permissions
- Camera and microphone permissions requested on first launch
- If denied, tapping "Record Swing" prompts you to open Settings

## How It Works

### Pose Detection During Recording
When you tap the record button, the app starts two things simultaneously:
1. **Video recording** — saves a `.mp4` file to app-private storage
2. **Camera image stream** — each raw camera frame is fed into [ML Kit Pose Detection](https://developers.google.com/ml-kit/vision/pose-detection) running entirely on-device

For every frame where a person is detected, 33 body landmarks are captured (each with x/y coordinates and a confidence score). These are stored as a timestamped JSON file (`poses.json`) saved next to the video file.

### Skeleton Overlay
On the Analysis screen, a `CustomPainter` reads the saved pose data and draws the skeleton on a transparent layer that sits directly over the video. As you scrub or play the video, a binary search finds the nearest captured pose frame and updates the overlay in real time.

Bones drawn (12 connections):
- Shoulders ↔ elbows ↔ wrists (both arms)
- Shoulders ↔ hips (torso sides)
- Left shoulder ↔ right shoulder, left hip ↔ right hip
- Hips ↔ knees ↔ ankles (both legs)

High-confidence joints (>50%) are drawn solid green; lower-confidence ones are drawn semi-transparent.

### Swing Phase Detection
The app uses a state machine to label each frame:

| Phase | Detection Logic |
|-------|----------------|
| **Setup** | Wrist near hip level, minimal movement |
| **Backswing** | Wrist rises above shoulder height |
| **Impact** | Wrist descends back through the ball zone (local minimum) |
| **Follow-Through** | Wrist rises again on the far side |

A 5-frame sliding window smooths noisy transitions.

### Joint Angle Math
- **Shoulder / Hip rotation** — `atan2(Δy, Δx)` of the line connecting the left and right landmarks, converted to degrees from horizontal
- **Elbow bend** — 3-point angle formula using dot products: `acos((BA · BC) / (|BA| × |BC|))` where B is the elbow, A is the shoulder, C is the wrist. 180° = fully straight arm.

### Coaching Tips (Rule-Based)
Tips are generated by comparing measured angles at key frames against known biomechanical targets:

| Condition | Tip |
|-----------|-----|
| Shoulder rotation < 30° at backswing peak | Rotate shoulders more — aim for 45°+ |
| Hip rotation > 25° at backswing | Restrict hips to build more torque |
| Trail elbow bend < 80° at backswing | Avoid "flying elbow" — keep it closer to body |
| Lead elbow bend < 150° at impact | Keep lead arm straighter at impact |
| Shoulder rotation < 10° at impact | Drive shoulder rotation through impact |

Up to 3 tips are shown, prioritised by severity.

## Downloading the APK

This repo has a GitHub Actions workflow that automatically builds an APK on every push.

**Steps:**
1. Go to the **Actions** tab on this repository
2. Click the most recent **"Build Android APK"** workflow run
3. Scroll to the **Artifacts** section at the bottom
4. Download **`golf-swing-analyzer-debug`** (for testing) or **`golf-swing-analyzer-release`** (for daily use)
5. Transfer the `.apk` file to your Android device
6. On your device, enable **"Install from unknown sources"** in Settings → Security (or Settings → Apps → Special app access on newer Android)
7. Open the `.apk` file to install

> The debug build includes logging and is easier to sideload. The release build is smaller and faster.

## Building Locally

Requirements: Flutter 3.10+, Android SDK, Java 17

```bash
# Clone and enter repo
git clone https://github.com/Killswit3h/Hole-In-One.git
cd Hole-In-One

# Install Flutter dependencies
flutter pub get

# Build debug APK (quick, good for testing)
flutter build apk --debug

# Build release APK (optimised)
flutter build apk --release

# APK output location:
# build/app/outputs/flutter-apk/app-debug.apk
# build/app/outputs/flutter-apk/app-release.apk

# Install directly to a connected device
flutter install
```
